# Homelab

GitOps-driven homelab running on Proxmox VE with a 3-node Kubernetes cluster managed by Flux CD.

## Stack

| Layer | Technology |
|---|---|
| Hypervisor | Proxmox VE |
| Container Orchestration | Kubernetes (3 nodes) |
| GitOps | Flux CD |
| Ingress | ingress-nginx |
| TLS | cert-manager + Let's Encrypt |
| Storage | Longhorn |
| Monitoring | kube-prometheus-stack (Prometheus + Grafana) |
| Media Server | Jellyfin |
| Dashboard | Homer |

## Repository Structure

```
.
├── infrastructure/
│   └── proxmox/
│       └── cloud-init/          # Cloud-init reference configs for VMs
└── kubernetes/
    ├── clusters/
    │   └── homelab/
    │       ├── flux-system/     # Auto-generated by flux bootstrap — do not edit
    │       ├── infrastructure.yaml   # Flux Kustomizations for infrastructure layer
    │       └── apps.yaml             # Flux Kustomization for apps layer
    ├── infrastructure/
    │   ├── controllers/         # Core controllers: cert-manager, ingress-nginx, longhorn
    │   └── configs/             # Cluster-wide configs: ClusterIssuers
    └── apps/
        ├── base/                # Base HelmRelease definitions
        │   ├── monitoring/      # kube-prometheus-stack (Prometheus + Grafana)
        │   ├── jellyfin/        # Media server
        │   └── homer/           # Homelab dashboard
        └── homelab/             # Cluster overlay — pulls in base apps
```

## How it works

Flux CD watches this repo on `main` and reconciles the cluster in dependency order:

```
infrastructure-controllers  →  infrastructure-configs  →  apps
(cert-manager, ingress,           (ClusterIssuers)        (monitoring,
 longhorn)                                                  jellyfin, homer)
```

Any `git push` to `main` is automatically applied to the cluster within minutes.

See [docs/kubernetes-setup.md](docs/kubernetes-setup.md) for Flux bootstrap instructions.

## Before going live

- Replace all `example.com` hostnames in HelmRelease files with your actual domain
- Add your email to `kubernetes/infrastructure/configs/cluster-issuers.yaml`
- Set your timezone in Jellyfin and Homer
- Update the Jellyfin `hostPath` to your actual media mount
- Encrypt secrets with SOPS before committing (see setup doc)
